# VitestのIn-Source Testing

VitestにはIn-Source Testingという機能があります。

https://vitest.dev/guide/in-source

これは、テストコードを実装のソースコード内に埋め込める機能です。
埋め込んだテストコードはビルド時にはTree Shakingされて削除されるので、本番ビルドには影響しません。
RustやDenoのテストのようですね。

自分はとても便利でよく使っています。

# TypescriptのSwitch文を型安全に書く

様々な記事で、TypescriptのSwitch文は、default節でnever型になっているかを確認することで型安全に書くことができると紹介されています。

```typescript

type Action = 'add' | 'remove' | 'update'

function reducer(action: Action) {
  switch (action) {
    case 'add':
      return 'add'
    case 'remove':
      return 'remove'
    case 'update':
      return 'update'
    default:
      const _: never = action // ← never型になっているかを確認。そうでなければtype error
      throw new Error('Invalid action')
  }
}
```

こうしたエラーを発見するには、`tsc`を実行することでエラーを発見します。
ですが、せっかくtestツールとしてVitestを使っているので、Vitestでエラーを発見したいですね。
(あと設定によっては地味に`_`が未使用の旨のinfo errorが発生することもあってちょっと気持ちが悪い)

# In-Source TestingとSwitch文を組み合わせる

早速コードを見てみましょう！

```typescript
type Action = 'add' | 'remove' | 'update'

function reducer(action: Action) {
  switch (action) {
    case 'add':
      return 'add'
    case 'remove':
      return 'remove'
    case 'update':
      return 'update'
    default:
      throw new Error('Invalid action')
      if(import.meta.vitest) {
        const { it, expect } = import.meta.vitest
        it('should never reach here', () => {
            expectTypeOf(action).toEqualTypeOf<never>()
            expect.unreachable("Invalid action")
        })
      }
 }
}
